查找到两个链表的第一个共同结点

一道不入流大学的数据结构练习题：

单链表A和单链表B有共同结点。试设计找出两个链表的第一个共同结点的算法，且该算法的时间复杂度为O(n+m)，其中n是链表A的长度，m是链表B的长度。

作为一个!acmer，看到这个问题的时候，第一反应就是暴力遍历，对每一个链表A的结点都对链表B进行遍历，直到找到共同结点。

但是很显然，这种暴力做法的算法复杂度是O(n*m)而不是O(n+m)。

容易发现，因为是单链表，所以链表A与链表B从第一个相同结点开始，之后的所有结点都会是相同的，也就是链表A与链表B的后x结点是相同的（从第一个相同节点到最后一个结点）。

所以，两个链表的长度差d实际上就是两个链表第一个共同结点前的的长度差。

因此，只要使较长的链表先下历d个结点，再对两个链表同时向下遍历，则必定会同时到达第一个共同结点，此时的算法复杂度为O(n+m)。

C的代码实现如下（链表带头结点）：

//获取链表长度
int getLength(Node *head)    
{
    int l=0;

    while(head->next)
    {
        head=head->next;
        l++;
    }
    return l;
}
//查找第一个共同结点
Node *Find(Node *Ahead, Node *Bhead)
{
    int n=getLength(Ahead);
    int m=getLength(Bhead);
    Node *p1,*p2;
    int d;

    //获取长度差d
    //获取较长链表(p1)，较短链表(p2)
    if(n>m)
    {
        p1=Ahead->next;
        p2=Bhead->next;
        d=n-m;
    }
    else
    {
        p1=Bhead->next;
        p2=Ahead->next;
        d=m-n;
    }
    //较长链表先下历d个结点
    for(int i=0;i<d;i++)
    {
        p1=p1->next;
    }
    //两个结点同时向下遍历，找到相同结点时退出并返回该结点
    while(p1 && p2)
    {
        if(p1==p2)
            return p1;
        p1=p1->next;
        p2=p2->next;
    }
    //找不到相同结点则返回NULL
    return NULL;
}
